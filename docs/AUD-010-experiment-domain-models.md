## Summary

Defines the SQLAlchemy ORM models for experiment tracking (`Experiment`, `Run`, `Document`, `Artifact`) and the `PipelineConfig` Pydantic schema used as the canonical representation of a pipeline configuration.

---

## What Was Implemented

### ORM Models (`api/db/models.py`)

#### `Experiment`

Represents a named, reproducible pipeline configuration. One experiment can have many runs.

| Column | Type | Notes |
|--------|------|-------|
| `id` | UUID | Primary key |
| `name` | String(255) | Unique — used as MLflow experiment name |
| `description` | Text | Optional human-readable description |
| `pipeline_config` | JSON | Full `PipelineConfig` dict — enables reconstruction |
| `created_at` | DateTime (UTC) | Auto-set on insert |

#### `Run`

A single execution of an experiment. Contains full lineage metadata.

| Column | Type | Notes |
|--------|------|-------|
| `id` | UUID | Primary key |
| `experiment_id` | UUID FK | Links to `experiments.id` |
| `mlflow_run_id` | String(64) | Links to MLflow run for metric retrieval |
| `git_sha` | String(40) | Git commit SHA at time of run |
| `branch` | String(255) | Git branch name |
| `timestamp` | DateTime (UTC) | Run start time |
| `status` | String(32) | `running \| completed \| failed` |
| `config_snapshot` | JSON | Full config at run time |
| `dataset_version` | String(64) | Version of dataset used (nullable) |

#### `Document`

An uploaded source document (PDF, DOCX, EPUB).

| Column | Type | Notes |
|--------|------|-------|
| `id` | UUID | Primary key |
| `original_filename` | String(512) | Original file name |
| `storage_key` | String(1024) | B2 object key |
| `content_type` | String(128) | MIME type |
| `size_bytes` | Integer | File size |
| `uploaded_at` | DateTime (UTC) | Upload timestamp |

#### `Artifact`

A file generated by a pipeline run, linked back to its run.

| Column | Type | Notes |
|--------|------|-------|
| `id` | UUID | Primary key |
| `run_id` | UUID FK | Links to `runs.id` |
| `artifact_type` | String(32) | `markdown \| audio \| metrics \| traces` |
| `storage_key` | String(1024) | B2 object key |
| `created_at` | DateTime (UTC) | Creation timestamp |

### `PipelineConfig` Schema (`shared/schemas.py`)

Pydantic model representing the full configuration for a pipeline run. Used both as API request schema and as the config stored in `Experiment.pipeline_config`.

```python
class PipelineConfig(BaseModel):
    pdf_parser: str = "docling"
    ocr_backend: str = "rapidocr"
    chunker: str = "paragraph"
    chunk_max_chars: int = 800
    tts_provider: str = "kokoro"
    tts_voice: str = "af_bella"
    tts_speed: float = 1.0
    llm_model: str = "qwen2.5:0.5b"
    llm_orchestration: str = "langgraph"
    sentence_detector: str = "spacy"
    g2p_backend: str = "espeak"
    dsp_preset: str = "default"
    extra: dict = {}
```

### Experiment YAML Format

Experiments can also be defined as YAML files stored in `experiments/`:

```yaml
# experiments/baseline-kokoro-v1.yaml
name: baseline-kokoro-v1
description: Paragraph chunker + Kokoro default voice
pipeline_config:
  pdf_parser: docling
  chunker: paragraph
  tts_provider: kokoro
  tts_voice: af_bella
  tts_speed: 1.0
  llm_model: qwen2.5:0.5b
```
